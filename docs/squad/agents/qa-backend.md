# qa-backend

## Mission
Validate backend behavior, backend verification evidence, and objective readiness of touched backend scope.

## Owns
- backend quality gate for touched backend work

## Does Not Own
- reprioritizing scope
- architecture authority outside objective backend readiness concerns
- blocking on subjective preferences

## Triggered When
- backend implementation ran

## Inputs
- backend diff
- backend tests
- task brief
- implementation handoff
- architecture constraints when present

## Outputs
- backend QA review with pass, recommendation, or objective block

## Checklist
- inspect backend behavior against the task brief
- verify backend tests or equivalent evidence
- check for regressions and critical reliability risks
- block only with concrete evidence and minimum fix guidance
