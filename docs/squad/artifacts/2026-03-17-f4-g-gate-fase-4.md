## Architecture Review

- Reviewer: `tech-lead`
- Scope reviewed:
  - agregacao do snapshot operacional do dashboard
  - contrato HTTP de `GET /api/dashboard`
  - contrato HTTP de notificacoes e historico
  - cliente frontend, mapper e renderizacao final na UI
- Boundaries checked:
  - `backend/app/application/dashboard.py` agrega `summary`, `events` e `notifications` sem depender de nova persistencia
  - `backend/app/api/routes/dashboard.py` expoe `GET /api/dashboard` e devolve `DashboardSnapshotResponse`
  - `backend/app/schemas/dashboard.py` fixa o payload snake_case para dashboard e notificacoes
  - `backend/app/api/routes/notifications.py` expoe listagem, dismiss individual e dismiss em lote via schemas dedicados
  - `backend/app/schemas/notification.py` centraliza o contrato de notificacao e normaliza datetimes UTC
  - `web/src/lib/api/dashboard.ts` consome apenas `GET /api/dashboard`
  - `web/src/entities/dashboard/model.ts` faz o mapeamento snake_case -> camelCase para `summary`, `events` e `notifications`
  - `web/src/features/dashboard/screens/dashboard-screen.tsx` usa apenas `loadDashboardSnapshot` para loading, erro, vazio, refresh e renderizacao completa
  - `web/src/features/notifications/components/notification-history.tsx` renderiza dados tipados e formatados, sem expor JSON cru
